¿Qué es XML?

El lenguaje de marcado extensible (XML) permite definir y almacenar datos de forma compartible. XML admite el intercambio de información entre sistemas de computación, como sitios web, bases de datos y aplicaciones de terceros. Las reglas predefinidas facilitan la transmisión de datos como archivos XML a través de cualquier red, ya que el destinatario puede usar esas reglas para leer los datos de forma precisa y eficiente.

A diferencia de otros lenguajes de programación, XML no puede realizar operaciones de computación por sí mismo. En cambio, se puede implementar cualquier software o lenguaje de programación para la administración estructurada de datos.

¿Qué es JSON?

JSON (JavaScript Object Notation) es un formato ligero de intercambio de datos. JSON es de fácil lectura y escritura para los usuarios. JSON es fácil de analizar y generar por parte de las máquinas. JSON se basa en un subconjunto del lenguaje de programación JavaScript, Estándar ECMA-262 3a Edición - Diciembre de 1999. JSON es un formato de texto completamente independiente del lenguaje, pero que utiliza convenios que resultan familiares a los programadores de lenguajes de la familia C, incluidos C, C++, C#, Java™, JavaScript, Perl, Python y mucho otros. Estas características hacen de JSON un lenguaje de intercambio de datos ideal.

SEMEJANZAS Son formatos autodescriptivos, es decir que se diseñaron para que fueran legibles para el ser humano y para la máquina.
Tienen un buen soporte para las definiciones de contenido y la validación.
JSON y XML son ampliamente utilizados, por lo que son soportados por muchos lenguajes de programación.
Pueden utilizarse para la serialización de datos (convertir una estructura de datos a un formato para su almacenamiento o transmisión). Esto significa que puedes utilizarlos para pasar información de una aplicación o sistema a otro a través de un canal de comunicación como HTTP o SOAP
Están basadas en texto.
Tienen estructuras jerárquicas en las que cada campo tiene su nombre y valor separados por comas.
DIFERENCIAS JSON es ligero y fácil de leer, pero no proporciona información sobre el esquema o el tipo. Es ideal para compartir datos entre varias aplicaciones. XML es legible por los humanos pero no necesariamente por las máquinas. Puede utilizarse para compartir información estructurada entre programas y documentos.
Puede utilizarse con archivos JavaScript o de texto plano. Sólo puede almacenarse como archivo de texto
Es una forma de almacenar datos de forma compacta para que los programas puedan leerlos. Es más fácil de escribir y leer, ya que utiliza menos caracteres. Es una forma específica de lenguaje de marcas para almacenar datos de forma organizada. Tiene más características que JSON, pero también es más complicado porque requiere más información sobre la estructura del documento antes de poder leerlo.
Sólo se admiten como tipos de datos la cadena, el número, el booleano y el nulo. Se pueden utilizar muchos otros tipos, como la fecha y la hora, para describir los datos de XML.
